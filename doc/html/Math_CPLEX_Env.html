NAME

Math::CPLEX::Env - Perl extension to create, modify, use, and close CPLEX environment objects.

SYNOPSIS

  use Math::CPLEX::Env;
  use Math::CPLEX::Base;

  # create new CPLEX environment object
  $cplex_env = Math::CPLEX::Env->new();

  # get CPLEX version
  $version = $cplex_env->version();

  # set a CPLEX double parameter (mip gap to 1e-7)
  $cplex_env->setdblparam(&Math::CPLEX::Base::CPX_PARAM_EPAGAP, 1e-7)

  # get a CPLEX double parameter
  $mip_gap = $cplex_env->getdblparam(&Math::CPLEX::Base::CPX_PARAM_EPAGAP);

  # set a CPLEX int parameter (number of used threads to 3)
  $cplex_env->setintparam(&Math::CPLEX::Base::CPX_PARAM_THREADS, 3)

  # get a CPLEX int parameter
  $num_threads = $cplex_env->getintparam(&Math::CPLEX::Base::CPX_PARAM_THREADS);

  # create a new optimization problem programming object
  $lp = $cplex_env->createOP();

  # write non-default CPLEX parameters to file
  $cplex_env->writeparams("/tmp/cplex_param.txt");

  # close CPLEX environment
  $cplex_env->close();

DESCRIPTION

Math::CPLEX::Env uses methods of the module Math::CPLEX::Base to communicate with the CPLEX C library. In general Math::CPLEX::Env allows to create a new CPLEX environment, to set and get environment parameters, and write the current settings to a file. The definition and optimization of the actual optimization problem is done via the Math::CPLEX::OP module. An instance of a Math::CPLEX::OP object is obtained via Math::CPLEX::Env's createOP() method.

EXPORT

None by default.

new

Create a new Math::CPLEX::Env object. Returns undef if create the CPLEX environment failes.

   $cplex_env = Math::CPLEX::Env->new();

close

Close CPLEX environment and free used CPLEX resources. The method close() frees all optimization problems that were created via createOP(). Returns undef if closing the CPLEX environment fails.

   $cplex_env->close();

createOP

Obtain a new Math::CPLEX::OP object which is used to set, modify, solve and remove a CPLEX optimization problem. Returns undef if creating the optimization problem fails.

   $op = $cplex_env->createOP();

version

Get CPLEX version. Returns a string contain the version if successful, otherwise undef.

   $version = $cplex_env->version();

setdefaults

Set all CPLEX parameters to their default values. Returns the CPLEX error code in the case of an error. Returns undef if successful.

   $cplex_env->setdefaults();

setdblparam

Set a double CPLEX parameter. Returns undef if successful, otherwise the CPLEX error code.

   $cplex_env->setdblparam(&Math::CPLEX::Base::CPX_PARAM_EPAGAP, 1e-7)

getdblparam

Get a double CPLEX parameter. Returns the retrieved parameter if successful. Returns undef in case of an error.

   my $dbl_param = $cplex_env->getdblparam(&Math::CPLEX::Base::CPX_PARAM_EPAGAP);

setintparam

Set an integer CPLEX parameter. Returns undef if successful, otherwise the CPLEX error code.

   $cplex_env->setintparam(&Math::CPLEX::Base::CPX_PARAM_THREADS, 3);

getintparam

Get an integer CPLEX parameter. Returns the retrieved parameter if successful. Returns undef in case of an error.

   my $num_threads = $cplex_env->getintparam(&Math::CPLEX::Base::CPX_PARAM_THREADS);

setlongparam

Set a long integer CPLEX parameter. Returns undef if successful, otherwise the CPLEX error code.

   $cplex_env->setlongparam(&Math::CPLEX::Base::CPX_PARAM_ITLIM, 1000000000000);

getlongparam

Get a long integer CPLEX parameter. Returns the retrieved parameter if successful. Returns undef in case of an error.

   my $max_iterations = $cplex_env->getlongparam(&Math::CPLEX::Base::CPX_PARAM_ITLIM);

setstrparam

Set a string CPLEX parameter. Returns undef if successful, otherwise the CPLEX error code.

   $cplex_env->setstrparam(&Math::CPLEX::Base::CPX_PARAM_WORKDIR, "/tmp");

getstrparam

Get a string CPLEX parameter. Returns the retrieved parameter if successful. Returns undef in case of an error.

   my $working_dir = $cplex_env->getstrparam(&Math::CPLEX::Base::CPX_PARAM_WORKDIR);

writeparams

Write all non-default parameters to a text file. writeparams takes as input the name of the file to be written. Returns 1 if successful, otherwise undef.

   $cplex_env->writeparams("/tmp/cplex_params.txt");

The content of the file /tmp/cplex_params.txt might looks as follows:

   \CPLEX Parameter File Version 12.5.0.0
   CPX_PARAM_ITLIM                  1000000000000
   CPX_PARAM_WORKDIR                "/tmp"
   CPX_PARAM_THREADS                3
   CPX_PARAM_EPAGAP                 1.00000000000000e-07

Note that all default parameters are not written to the file.

CPLEX constants

CPLEX heavily uses constants, e.g. for setting and getting parameters and for evaluating the status of an optimization. Math::CPLEX allows to access these constants via methods that are provided by an AUTOLOAD function of the module Math::CPLEX::Base. Hence, constants can be accessed as follows:

   my $val_unbounded  = &Math::CPLEX::Base::CPX_INFBOUND;
   my $opti_mip_state = Math::CPLEX::Base::CPXMIP_OPTIMAL();

SEE ALSO
